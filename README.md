# VaultofCodes Internships Page (Minimal-JS Version)

A lightweight internship listing page rebuilt to use as little JavaScript as possible, while keeping search, filtering, and a tilt-on-hover effect on the cards.

## Files

| File | Purpose |
|---|---|
| `index.html` | Page markup. All 5 internship cards are written directly as static HTML (no JS templating/rendering). |
| `style.css` | All visual styling — layout, colors, animations, responsive rules. |
| `script.js` | The only JavaScript in the project (~90 lines). Handles filtering, search, the accordion sidebar, back-to-top/scroll progress, and card tilt. None of the Animations |
| `logo.jpg` | Background image used by `style.css`. |

Keep all four files in the same folder — `index.html` references the other three by relative path.

## What changed from the original

- Cards are hardcoded in HTML instead of being generated by JS from a data array.
- Removed: skeleton loading screen, custom cursor (dot/ring), `IntersectionObserver` scroll-reveal animation, and the requestAnimationFrame tilt/lerp loop.
- Kept, but simplified: filtering, search, accordion toggle, back-to-top button, and tilt-on-hover (now a direct transform on `mousemove` instead of an animation loop).

## Why JavaScript is still needed

**HTML and CSS can describe what the page looks like, but only JavaScript can make it respond to what the user does *after* the page has loaded.** Filtering internships by provider/fee/mode, live search-as-you-type, opening/closing the filter accordion, tracking scroll position for the progress bar and back-to-top button, and tilting a card toward the cursor all require reacting to real-time events (clicks, typing, mouse movement, scrolling) and updating the DOM accordingly — none of which CSS or HTML alone can do. **The goal here wasn't to remove JavaScript entirely, but to keep only the JavaScript that earns its place: the minimum needed to make the page interactive, while letting CSS handle everything about appearance and static structure that it's already capable of.**
